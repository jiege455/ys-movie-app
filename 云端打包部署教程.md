## 一、系统架构说明

```
用户操作（MacCMS后台）
    ↓ 上传图标，选择平台
你的服务器（PHP接口）
    ↓ 创建GitHub仓库，上传配置
GitHub Actions（云端构建）
    ↓ 下载源码，构建APK/IPA
GitHub Release（产物存储）
    ↓ 用户下载安装包
```

## 三、详细部署步骤

### 第1步：创建GitHub Token

1. 登录 [github.com](https://github.com)
2. 点击右上角头像 → **Settings**
3. 左侧最下方 → **Developer settings**
4. **Personal access tokens** → **Tokens (classic)**
5. 点击 **Generate new token (classic)**
6. 填写信息：
   - Note: `CloudBuild`
   - Expiration: 选择 `No expiration`（永不过期）或自定义时间
   - 勾选权限：
     - ✅ `repo`（完整仓库控制）
     - ✅ `workflow`（Actions工作流）
7. 点击 **Generate token**
8. **立即复制Token**（页面关闭后无法再次查看）

⚠️ **重要**：这个Token相当于你的GitHub密码，不要泄露给任何人！

---

### 第2步：创建GitHub模板仓库

1. 登录GitHub，点击右上角 **+** → **New repository**
2. 填写信息：
   - Repository name: `flutter-build-template`
   - Description: `云端打包模板仓库`
   - 选择 **Public**（公开仓库Actions免费）
   - ✅ 勾选 **Add a README file**
3. 点击 **Create repository**

#### 上传工作流文件

1. 进入刚创建的仓库
2. 点击 **Add file** → **Create new file**
3. 文件路径填写：`.github/workflows/cloud_build.yml`
4. 把 `cloud_build.yml` 文件内容复制进去
5. 点击 **Commit new file**

---

3. 等待打包完成
4. 生成的文件：`flutter_app_source.zip`

### 第5步：创建数据库表

1. 登录宝塔面板 → 数据库
2. 找到你的MacCMS数据库
3. 点击 **管理**（进入phpMyAdmin）
4. 点击 **SQL** 标签
5. 粘贴 `cloud_build.sql` 的内容
6. 点击 **执行**

或者使用命令行：

```bash
mysql -u你的用户名 -p你的密码 你的数据库名 < cloud_build.sql
```

## 五、常见问题

### Q1: 构建失败怎么办？

1. 查看GitHub Actions日志
2. 检查GitHub Token是否有效
3. 检查源码包是否能正常下载
4. 检查CMS配置是否正确

### Q2: iOS构建额度不够怎么办？

- 每月2000分钟免费额度
- 可以只构建Android（无限免费）
- 或购买额外额度

### Q3: 源码安全吗？

- ✅ 源码不会暴露在GitHub
- ✅ 构建完成后自动删除
- ✅ 下载接口有Token鉴权

### Q4: 可以修改默认配置吗？

可以，编辑 `cloud_build.php` 中的默认值：

```php
$appName = !empty($cmsConfig['app_name']) ? $cmsConfig['app_name'] : '你的默认名称';
$packageName = !empty($cmsConfig['package_name']) ? $cmsConfig['package_name'] : '你的包名';
```
